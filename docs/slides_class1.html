Exploratory versus explanatory

  • Exploratory: examine the structure of your data.

  • Explanatory: tell a story with your data.

Resources: clauswilke.com (telling a story and making a point), susielu.com (different interpretation of exploratory/explanatory).

Exploratory in the sense of data exploration. Exploratory in the sense of visualization exploration (as in previous example, where the same data can be plotted in several ways). In both cases, you can create ugly plots.

  • Is it explaining data?

  • Is the information complete and correct?

  • Are axes correct? (+ Should they have a zero-point?)

  • Do the colors work?

  • Are all elements needed?

  • What is the data to ink ratio?

  • Is it understandable & simple?

Font embedding

Vector images pick the closest font available (if the actual font is not available on the recipients computer). You can embed fonts into the vector image.

  • showtext converts characters into polygons. Embedding is no longer required, but text is no longer searchable.
  • extrafont supports font embedding, but supports fewer fonts.

Adobe Acrobat (paid version) can be used to manually embed fonts in a PDF.

Final assignment

Project 1

Ability development

  • Audience: Teachers / Prowise team
  • Changes in [score/ability] over [time/sessions/items] per [player/class/game/learning goal/time-period]
    • Prowise example: “How do first-time players improve in the first 10 sessions of the game [Verhaaltjes]?”
    • Teacher example: “Which children are not showing the expected improvement?” [decline / stopped growth]

Project 2

Teacher dashboard

  • Audience: Teachers

  • Make an overview of the scores of children in a classroom, for instance:

    • Which games were played in the last [week/month]?

    • How much did they play?

    • What are their ability scores?

  • Example: Overview that helps teachers to start off the day

  • Example: Overview that dives deeper into the scores of a [child/class]

Project 3

User engagement

  • Audience: Prowise team / Business Information

  • Can behavioral patterns in the use of the system be identified? For example:

    • How many children play in our system per [week/month]? Any differences between the three applications [Math Garden, Language Sea, Words&Birds]?

    • How many items have been completed per domain and/or learning goal in a [week/month/year]?

    • Are there differences in when children play [school hours, evening, weekend]?

    • How is the show coins/difficulty/question mark feature used?

Project 4

Motivation

  • Audience: Teacher / Prowise team / Data Scientists or researchers

  • Pick one or two motivational variables and compute them from log record data.

    • Examples of motivational metrics could be: [quitting, persistence, game / difficulty selection, number of items played per login], etc …

  • Are there [between- / within- player] differences in motivation? For example:

    • Do children differ in how many items they complete in a given session?

    • Do children consistently play the same games?

    • Does difficulty level predict quitting?

  • Teacher example: Which students are ‘highly’ or ‘poorly’ motivated/show undesired play behavior?

  • Researcher example: Are motivation effects stable across [time/games/age groups]?

Project 5

  • Audience: Prowise team / Data scientist

  • Investigate the scores [expected / observed] per [item/user/groups] within psychometric models. For example:

    • How fast are the [expected_scores] converging to [observed_scores]? Show the relation with playing frequency

    • Different categories of model misfits (e.g., discrepancies between [expected & observed scores per item/user]

    • Investigate item drift
